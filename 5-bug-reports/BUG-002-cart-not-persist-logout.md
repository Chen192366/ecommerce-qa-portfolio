# BUG-002 — 登出后购物车数据未持久化

## 基本信息

| 字段 | 值 |
|---|---|
| Bug ID | BUG-002 |
| 模块 | 购物车 (Cart) |
| 严重等级 | Major |
| 优先级 | P1 |
| 状态 | Open |
| 报告日期 | 2026-07-10 |
| 报告人 | Chen192366 |
| 环境 | Windows 11 / Chrome 最新版 / SauceDemo |

## 复现步骤

1. 使用 standard_user 登录
2. 添加 2 件商品到购物车
3. 验证购物车中有 2 件商品
4. 点击菜单 → Logout 登出
5. 重新使用 standard_user 登录
6. 打开购物车

## 预期结果

购物车中仍然保留之前添加的 2 件商品（购物车数据与用户账号关联）。

## 实际结果

购物车为空，之前添加的商品全部丢失。

## 截图

> 参见 `../6-screenshots/BUG-002-cart-persistence.png`

## 影响范围

- 真实电商场景下，用户登出后重新登录发现购物车被清空，严重影响用户体验
- 可能导致用户流失和订单损失
- 对多设备/多会话场景完全不支持

## 建议修复

1. 将购物车数据与用户 ID 关联，持久化到数据库或服务端 Session
2. 在客户端使用 Cookie 或 LocalStorage 作为降级方案
3. 在登出前给用户提示："登出后购物车将被清空，确认登出？"
