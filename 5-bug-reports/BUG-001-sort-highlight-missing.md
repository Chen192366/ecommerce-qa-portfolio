# BUG-001 — 商品排序下拉框未高亮当前选中项

## 基本信息

| 字段 | 值 |
|---|---|
| Bug ID | BUG-001 |
| 模块 | 商品列表 (Inventory) |
| 严重等级 | Minor |
| 优先级 | P2 |
| 状态 | Open |
| 报告日期 | 2026-07-10 |
| 报告人 | Chen192366 |
| 环境 | Windows 11 / Chrome 最新版 / SauceDemo |

## 复现步骤

1. 使用 standard_user 登录
2. 在商品列表页，默认排序为 "Name (A to Z)"
3. 点击排序下拉框，选择 "Price (high to low)"
4. 商品按价格降序排列
5. 再次点击排序下拉框查看

## 预期结果

排序下拉框中，当前生效的排序选项 "Price (high to low)" 应高亮显示或标记为选中状态。

## 实际结果

排序下拉框中没有任何选项被高亮标记。用户无法从 UI 直观判断当前使用何种排序，只能通过商品列表顺序推断。

## 截图

> 参见 `../6-screenshots/BUG-001-sort-highlight.png`

## 影响范围

- 用户体验降低，用户无法快速确认当前排序方式
- 频繁切换排序时容易困惑

## 建议修复

在排序下拉框 `<option>` 元素上添加 `selected` 属性，或在 UI 上增加当前排序的文本提示。
