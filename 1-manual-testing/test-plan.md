# 测试计划 — SauceDemo 电商平台

## 1. 测试范围

### 包含 (In Scope)
- 登录模块（标准用户 / 问题用户 / 锁定用户 / 性能测试用户）
- 商品列表页（菜单导航、排序功能、商品卡片展示）
- 商品详情页（图片、描述、价格、加入购物车）
- 购物车（添加/移除商品、继续购物、结算跳转）
- 结算流程（填写收货信息、订单确认、完成页）

### 不包含 (Out of Scope)
- 性能 / 压力测试（仅功能验证）
- 浏览器兼容性测试（仅 Chrome）
- 移动端适配测试
- 支付网关对接测试
- 安全性渗透测试

## 2. 测试策略

| 测试类型 | 方法 | 工具 |
|---|---|---|
| 功能测试 | 黑盒：等价类 + 边界值 + 场景法 | 手工执行 |
| 回归测试 | 核心流程自动化回归 | Selenium + Pytest |
| 接口测试 | HTTP 状态码 + 响应体字段断言 | Requests + Pytest |
| 数据验证 | 数据完整性 / 业务逻辑 SQL 审计 | MySQL |
| 缺陷管理 | 发现 → 记录 → 跟踪 → 验证关闭 | Markdown Bug 报告 |

## 3. 测试环境

| 项目 | 配置 |
|---|---|
| 操作系统 | Windows 11 |
| 浏览器 | Chrome (最新稳定版) |
| 被测地址 | https://www.saucedemo.com/ |
| 数据库 | MySQL 8.0.34 (本地) |
| 测试框架 | Pytest 9.x |
| 自动化 | Selenium 4.x + ChromeDriver |

## 5. 进度计划

| 阶段 | 内容 | 预计耗时 |
|---|---|---|
| Day 1 | 需求分析 + 编写测试计划、用例 | 0.5 天 |
| Day 1-2 | SQL 数据库搭建与数据验证 | 1 天 |
| Day 2 | API 接口测试 | 0.5 天 |
| Day 2-4 | Selenium POM 自动化框架 + 用例编写 | 2 天 |

## 6. 风险

| 风险 | 缓解措施 |
|---|---|
| 被测站点不可用 | 使用本地替代或等待恢复后继续 |
| 页面结构变更 | POM 设计模式降低维护成本 |
